← 接入层（用户感知面）
对话平面 · 同步 · 实时响应
Conversation Manager
始终在线，毫秒级响应。用户的问题、打断、指令都在这里处理。无论后台有多少任务在跑，这一层永远不阻塞。
💬Session Handler
管理当前对话上下文，处理多轮问答，维持用户感知的连续性
🎯Intent Classifier
判断用户输入是：新任务指令 / 对话问答 / 对进行中任务的干预 / 取消/暂停指令
🔔Notification Broker
任务主动问用户时的桥梁，向 Mobile 推送 push notification，等待用户响应后唤醒等待中的任务
🎙️Voice Pipeline
STT → 声纹识别(身份) → 意图理解 → TTS。生物识别在这一层完成，权限决策前置
任务平面 · 异步 · 持久执行
Task Orchestrator
独立于对话运行。每个 Task 有自己的状态机、检查点、资源配额。崩溃可以从检查点恢复，跨设备可见。
🧩Task Planner
接收高层目标，分解为有依赖关系的子任务 DAG。可以 re-plan（遇到障碍时重新规划）
⚙️Task Executor
串行/并行执行子任务，每步完成后写入检查点。超时/失败自动重试，超限才上报用户
💾Task Store
持久化每个 Task 的状态、计划、执行轨迹、检查点。进程重启后可从此恢复
🏭 Dynamic Tool Factory 新概念核心能力
当 Executor 发现任务需要的工具不存在时，唤醒 Code Agent 生成并沙箱测试一个新工具/脚本。成功后可选择"临时使用"或"提升为永久工具"注册到 Capability Bus
⚡ 打断 / 上下文切换
共享基础层（两个平面共用）
🧠
Memory System
工作记忆 + 情景记忆 + 语义记忆(RAG)。跨会话、跨任务共享，向量检索
🔌
Capability Bus
内置工具 + MCP 客户端 + Skill Registry + 动态注册工具。统一调用接口
⏰
Trigger Engine
Cron 调度 + 事件驱动 + 条件触发。主动唤醒 Jarvis，无需用户发起
🛡️
Identity & Policy
多模态生物识别 + 权限分级。谁能调用什么能力，每个动作都过策略
📦
Sandbox Runtime
动态生成的代码 + 外部工具的执行环境。资源限制、隔离、审计
📡
Event Bus
两个平面之间 + 所有层之间的事件传递。打断/恢复信号都走这里
🗄️
Persistent Store
SQLite / PostgreSQL。任务状态、事件日志、工具注册表、用户配置
🔭
Observability
每次 run 可回放、可分析。训练数据来源：什么决策是好的，什么需要干预

💡 架构的核心思想：双平面设计
对话平面是实时的、以用户为中心的、同步的；任务平面是异步的、以目标为中心的、持久的。
两者通过 Event Bus 解耦，打断信号是事件，恢复信号也是事件，任何一边都不直接调用另一边的内部方法。
这是现有 openjax 架构里没有的东西，也是新版本最值得从头设计的核心部分。
