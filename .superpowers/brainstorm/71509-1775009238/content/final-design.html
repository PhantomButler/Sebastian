三大扩展点（新能力 = 新文件，不改核心）
🔧 新增工具
1在 capabilities/tools/ 创建 my_tool.py
2实现 @tool 装饰器接口
3启动时 Capability Bus 自动扫描注册
4所有 Agent 立即可用
🤖 新增 Sub-Agent
1在 agents/ 创建新目录
2写 manifest.toml（能力声明 + 专属工具 + 知识库 + Prompt）
3可选：写 agent.py 覆盖特定行为
4Jarvis 启动扫描，A2A 自动可达
⏰ 新增触发器
1在 trigger/ 创建 trigger 配置
2声明触发条件（cron / 事件 / 条件）
3绑定目标 Agent 和任务模板
4Trigger Engine 自动调度执行
关键设计决策
主语言？
Python 3.12+，生态最适合 AI + ML + 生物识别
Agent 通信协议？
A2A（Google 开放规范），任务委派/上报/升级
工具接入协议？
MCP（Anthropic），所有 Agent 共用 MCP Client
Sub-Agent 扩展方式？
manifest.toml + 目录扫描，不改核心代码
任务 vs 会话？
Task 是一等公民，跨 turn/跨会话，有检查点
双平面设计？
对话平面永不阻塞，任务平面异步持久运行
动态工具创建？
Code Agent 写代码 → 沙箱测试 → 注册到 Capability Bus
部署方式？
Docker（解决 Python 单二进制问题）+ docker-compose 一键启动
记忆系统？
工作记忆(内存) + 情景记忆(SQLite) + 语义记忆(ChromaDB RAG)
现有 openjax？
保留参考，新项目独立 repo，从 0 开始 Python 实现
