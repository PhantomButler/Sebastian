💡 Sub-Agent 即 MCP Server 的好处
新增一个 Sub-Agent = 创建一个 MCP Server 项目，声明它的 tools/resources/prompts。Jarvis 启动时自动扫描注册，无需修改核心代码。

不同 Sub-Agent 可以用不同语言实现（Stock Agent 用 Python，因为有量化库；Life Agent 用 Go，因为需要稳定运行）。MCP 协议是语言无关的。
⚡ 并行任务的关键约束
并行不是无限的。需要一个 Resource Budget：同时最多跑几个 LLM 调用？API 成本控制在哪里？哪些任务优先级更高？

建议：Jarvis 维护一个全局 Concurrency Governor，各 Sub-Agent 也有自己的内部并发上限，互不干扰但受全局管控。
