# Contributing to Sebastian

Thanks for your interest in contributing! This guide covers setting up the development environment, running checks, and the PR workflow.

## Development Environment

### Prerequisites

- Python 3.12+
- Android Studio (for mobile development)
- Node.js (for iOS / React Native builds)

### Backend Setup

```bash
git clone https://github.com/Jaxton07/Sebastian.git
cd Sebastian
pip install -e ".[dev,memory]"
sebastian serve
```

Isolated dev environment (separate data dir, different port):

```bash
./scripts/dev.sh    # Data dir: ~/.sebastian-dev, Port: 8824
```

### Android App from Source

```bash
cd ui/mobile-android
echo "sdk.dir=$HOME/Library/Android/sdk" > local.properties
./gradlew installDebug
```

Full setup instructions: [ui/mobile-android/README.md](ui/mobile-android/README.md).

### iOS from Source

No pre-built iOS binary is distributed. Developers can build from source:

```bash
cd ui/mobile
npm install --legacy-peer-deps
npx expo run:ios    # Requires macOS + Xcode
```

## Development Commands

```bash
# Backend tests
pytest

# Lint
ruff check sebastian/ tests/
ruff format sebastian/ tests/

# Type checking
mypy sebastian/

# Android unit tests
cd ui/mobile-android && ./gradlew test
```

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `SEBASTIAN_OWNER_NAME` | — | Owner display name (set during first-launch wizard) |
| `SEBASTIAN_GATEWAY_HOST` | `127.0.0.1` | Gateway bind address |
| `SEBASTIAN_GATEWAY_PORT` | `8823` | Gateway bind port |
| `SEBASTIAN_DATA_DIR` | `~/.sebastian` | Data directory |

> **Note**: From v0.2.0, Owner account, JWT signing key, and LLM API keys are **not** configured via environment variables:
> - Owner account is stored in `<data_dir>/sebastian.db` (`users` table)
> - JWT key comes from `<data_dir>/secret.key` (auto-generated by setup wizard)
> - LLM API keys are configured via the Android app (Settings → Providers) and stored with Fernet encryption

## Branch Model & PR Workflow

- `main`: Protected branch. Only accepts PR squash merges. Release baseline.
- `dev`: Active development branch. Daily work goes here.

### Before starting work

```bash
git checkout dev
git fetch origin main
git rebase origin/main              # Squash-merged commits are auto-skipped
git push --force-with-lease         # Flatten dev onto main
```

### Commit conventions

- Use `git add <specific-files>` — never `git add .` or `git add -A`
- Format: `type(scope): summary` (e.g. `feat(gateway): add streaming support`)
- Types: `feat` / `fix` / `docs` / `refactor` / `chore` / `test` / `style` / `ci`
- Keep commits atomic — one commit, one concern

### Creating a PR

```bash
# Push to dev (already tracking origin/dev)
git push

# Create PR targeting main
gh pr create --base main --head dev
```

PR requirements:
- CI checks pass: `backend-lint`, `backend-type`, `backend-test`, `mobile-lint`
- 1 approval
- Squash merge into `main`

After merge, re-sync `dev` following the "Before starting work" steps above.

## Code Style

- Python 3.12+ with `from __future__ import annotations`
- 4-space indentation, PEP 8 naming
- Type annotations on all public and internal functions (including `-> None`)
- Use `str | None` union syntax (not `Optional`)
- Single file: recommended ≤500 lines, hard limit 800 lines
