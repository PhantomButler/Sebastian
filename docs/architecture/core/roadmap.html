Reality vs Future 当你连续看 spec、README 和代码时，这页用来把“已实现”和“未来设计”重新切开。

Phase 1 已经是现实的部分

  • Gateway：REST、JWT、SSEManager、session 级 stream。
  • Core Runtime：BaseAgent streaming、AgentLoop generator、AgentPool、TaskManager。
  • Store：Session agent_type/agent_id、文件系统 Session / Task / Checkpoint 真相。
  • App 骨架：Chat / SubAgents / Settings，基础联调链路可用。

Phase 1 仍可继续完善

  • App 端更强的 block 级呈现和督导视图。
  • Task 细节可视化与 approval 体验打磨。
  • 围绕现有 runtime 契约继续产品化，而不是重写基础边界。

Phase 2 才应真正进入的系统

  • A2A 同进程协作、多 Agent 自动注册。
  • working / episodic / semantic memory 主干。
  • Code Agent 与 Dynamic Tool Factory。

Phase 3-4 的体验与主动性扩展

  • 语音链路、iOS、通知体系增强。
  • Trigger Engine 让 Sebastian 开始主动运行。
  • 更多 Sub-Agent 与更强 Web 管理界面。

Phase 5 的真正安全边界

  • 声纹、人脸、活体检测、多因素身份。
  • Owner / Family / Guest 权限与抗仿冒。
