Core Runtime Focus 这一页只回答一件事：当前仓库里的运行时核心边界到底怎么切。

Boundary Matrix

先把每个核心模块的一句话职责钉死，再去读它们的代码细节。

BaseAgent
副作用边界。处理历史、publish、工具执行、partial 保存与 turn 收口。
AgentLoop
纯流式生成边界。负责把 LLM 输出统一成结构化事件，不做持久化和 EventBus 发布。
stream_events
BaseAgent 与 AgentLoop 之间的类型契约，也是 block 级 UI 的语义基础。
TaskManager
后台 task 生命周期和合法状态转换的集中入口。
AgentPool
worker 身份与并发槽位管理，不是通用任务队列替代品。
SessionStore
当前文件系统真相来源，Session / Task / Checkpoint 都要回到这里。

Current Runtime Dependency Shape

这不是完整类图，而是最值得记住的依赖方向。

Sebastian 继承 BaseAgent，站在 orchestrator 层调用 runtime。
BaseAgent 持有 AgentLoop、SessionStore、EventBus、CapabilityRegistry。
AgentLoop 向上产出 LLMStreamEvent，向下依赖 LLM stream 和 tool specs。
CapabilityRegistry 承接原生工具和 MCP 工具调用。
SessionStore 承接消息、任务与 checkpoint 持久化。
EventBus → SSE 把 runtime 事件送出到 App 观察面。
