Turn Data Flow

当前仓库里最核心的主路径。理解这条线后，很多局部代码会自动归位。

1. Mobile 发送用户消息，命中 POST /api/v1/turns。 ui/mobile → turns API
2. Gateway 立即返回 session_id，不等待 LLM 完成。 gateway routes
3. Sebastian / BaseAgent 发布 turn.received，加载历史，启动 streaming task。 base_agent.py
4. AgentLoop 生成 thinking / text / tool 的结构化事件流。 agent_loop.py
5. Event + Store partial / final 内容落盘，事件进入 EventBus。 session_store.py
6. SSE → App 前端根据 block 级事件恢复界面与状态。 sse.py

Interrupt Workflow

当前设计不是 fork，也不是 ghost branch，而是明确的 cancel + partial + 新上下文继续。

旧 stream 正在输出 _active_stream 尚未结束。
新消息到达 BaseAgent 取消当前 stream task。
保存 partial 写入 assistant partial，并发出 turn.interrupted。
重新开始 用新的上下文发起下一个 turn。

Task Workflow

Task 不是“聊天的附属物”，而是 Session 里的持久执行单元。

CREATED 提交到 TaskManager.submit()。
PLANNING Phase 1 中主要作为预留态，当前大多直通 RUNNING。
RUNNING 后台执行，伴随 task.* 事件和 Session 计数变化。
TERMINAL 进入 COMPLETED / FAILED / CANCELLED 之一。

Worker Acquire / Release

AgentPool 的意义不是“多开”本身，而是稳定 worker 身份与并发槽位。

Acquire 优先拿空闲 worker，例如 stock_03。
No idle worker 创建 waiter，进入 queue。
Release 有 waiter 时，优先直接转交，而不是先回 idle。
Status visible Gateway / App 可以观察具体 worker 状态。
