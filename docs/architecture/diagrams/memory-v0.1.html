① 三层记忆总览

三层记忆各有作用域和生命周期。已实现 WorkingMemory 和 EpisodicMemory 已有， 新增 SemanticMemory 是本次要建设的核心。

设计原则：不引入新基础设施依赖。个人规模（单 Owner，数百到数千条记忆）SQLite FTS5 全文检索毫秒级响应，向量 DB / PostgreSQL 均属过度设计。

② 数据模型

memories 表存储语义记忆，memories_fts 虚拟表提供全文索引。 合并了 Kimi 方案中 entity_key/value 结构化字段和 confidence/source/expires_at 的优良设计。

字段类型说明
id TEXT PK UUID，唯一标识
category TEXT preference / fact / project / habit / constraint
entity_key TEXT 结构化键，如 communication_style / current_project。可为空（自由文本记忆）
entity_value TEXT 结构化值，如 简洁中文。与 entity_key 配对
content TEXT NOT NULL 自然语言描述，用于 FTS5 检索和注入 system prompt
source TEXT explicit（用户主动说）/ inferred（Agent 从对话提取）
confidence REAL 0.0–1.0。explicit 固定 1.0，inferred 由 LLM 提取时给出
expires_at DATETIME 可选。临时偏好用，如"本周我在出差"。NULL 表示永久有效
source_session_id TEXT 来源 session，用于追溯
created_at DATETIME 创建时间
updated_at DATETIME 最后更新时间（upsert 时更新）
last_accessed_at DATETIME 最近被检索注入的时间，用于排序权重
access_count INTEGER 被检索注入的次数，越高优先级越高

Upsert 冲突策略：同一 entity_key 收到新值时，若新值 confidence > 旧值 confidence × 0.8，则覆盖更新；否则保留旧值。explicit 来源的写入永远覆盖 inferred。

③ 写入流水线

两条写入路径：即时写入（用户主动触发，精准高置信）和 Session 沉淀（后台异步，从对话中推断提取）。

flowchart TD subgraph PATH_A["路径 A — 即时写入（用户主动）"] U1["用户说: 记一下我喜欢简洁回复"] A1["Agent 识别意图\n调用 memory_save 工具"] A2["memory_save\ncategory=preference\nentity_key=communication_style\ncontent=喜欢简洁回复\nsource=explicit\nconfidence=1.0"] A3["直接写入 memories 表\n同步更新 FTS5 索引"] U1 --> A1 --> A2 --> A3 end subgraph PATH_B["路径 B — Session 沉淀（后台异步）"] B1["Session 状态变为 idle 或 stalled"] B2["EventBus 发布\nSESSION_IDLE 事件"] B3["ConsolidationWorker\n订阅事件，异步触发"] B4["读取该 session\n完整对话历史"] B5["调用 LLM\n结构化提取事实\n返回 JSON 列表"] B6["逐条 Upsert\nsource=inferred\nconfidence=LLM 给出"] B7["写入 memories 表\n更新 FTS5 索引"] B1 --> B2 --> B3 --> B4 --> B5 --> B6 --> B7 end style PATH_A fill:#1a2a1a,stroke:#3a7a3a style PATH_B fill:#1a1a2a,stroke:#3a3a7a style A2 fill:#1a3a1a,stroke:#3a8a3a style B5 fill:#2a1a2a,stroke:#7a3a7a
ConsolidationWorker 触发时机：复用现有 StalledWatchdog 和 Session 状态机。 Session 进入 idle（用户主动 stop）或 stalled（5min 无活动）时， 通过 EventBus 触发，不新增定时器。沉淀完成后给 session 打 consolidated=True 标记，避免重复提取。
不做每 turn 提取：每轮对话信息噪音大，额外 LLM API 费用高。只在 session 自然结束后做一次整体提取，性价比更高。

④ 读取流水线

被动注入（每次 turn 自动）和主动查询（Agent 按需）协作，覆盖高频常识和低频深查两种场景。

flowchart TD subgraph PASSIVE["被动注入（每次 turn 自动触发）"] P1["_stream_inner() 开始"] P2["_memory_section()\n查 memories 表\nFTS 匹配当前 session goal\n+ access_count + recency 排序\n取 top-8 条"] P3["注入 system prompt\n## What I know about you\n- 你喜欢简洁中文回复\n- 当前项目：Sebastian AI 管家\n..."] P4["同时更新\naccess_count +1\nlast_accessed_at"] P1 --> P2 --> P3 P2 --> P4 end subgraph ACTIVE["主动查询（Agent 按需调用工具）"] Q1["用户: 我之前说的那个出行计划\n是什么来着?"] Q2["Agent 判断需要深查"] Q3["调用 memory_search\nquery=出行计划"] Q4["FTS5 全文检索\nmemories_fts MATCH query\n返回相关记忆列表"] Q5["Agent 把结果\n带入回复上下文"] Q1 --> Q2 --> Q3 --> Q4 --> Q5 end style PASSIVE fill:#1a2a1a,stroke:#3a7a3a style ACTIVE fill:#1a2a2a,stroke:#3a7a7a style P3 fill:#1a3a1a,stroke:#3a8a3a
被动注入的位置：与 _session_todos_section() 相同模式，在 _stream_inner() 开始时动态拼接进 effective_system_prompt，每次 turn 刷新，不用 build_system_prompt()（那是静态的，重启才刷新）。

⑤ 与 Agent Loop 集成

最小侵入原则：只在现有流程的 4 个位置加 hook，不改 AgentLoop 核心逻辑。

sequenceDiagram participant U as 用户 participant BA as BaseAgent participant SM as SemanticMemory participant AL as AgentLoop participant EB as EventBus participant CW as ConsolidationWorker U->>BA: run_streaming(message, session_id) note over BA: Hook 1 — 读取记忆注入 prompt BA->>SM: top_memories(session_goal, limit=8) SM-->>BA: memories[] BA->>BA: effective_system_prompt =\nsystem_prompt + memory_section + todos_section BA->>AL: loop.stream(effective_system_prompt, messages) loop Agent Loop 多轮工具调用 AL-->>BA: TextDelta / ToolCallReady alt ToolCallReady == memory_save note over BA: Hook 2 — 即时写入 BA->>SM: upsert(source=explicit, confidence=1.0) else ToolCallReady == memory_search note over BA: Hook 3 — 主动查询 BA->>SM: search(query) SM-->>BA: results[] end end BA->>BA: TurnDone — 持久化对话 note over EB,CW: Hook 4 — Session 沉淀（异步） EB->>EB: 发布 SESSION_IDLE / SESSION_STALLED EB->>CW: 触发 ConsolidationWorker CW->>BA: 读取 session 完整历史 CW->>CW: LLM 提取结构化事实 CW->>SM: upsert(source=inferred, confidence=llm_score)
Hook 位置文件改动说明
1 — 读取注入 core/base_agent.py
_stream_inner()		 仿 _session_todos_section()，加 _memory_section() 拼入 prompt
2 — 即时写入 capabilities/tools/memory_save.py 新增 @tool，PolicyGate 注册，Agent 调用即写
3 — 主动查询 capabilities/tools/memory_search.py 新增 @tool，FTS5 查询，返回 ToolResult
4 — 沉淀触发 memory/consolidation.py 新增 ConsolidationWorker，订阅 SESSION_IDLE / SESSION_STALLED 事件

⑥ 新增工具一览

4 个新 Tool，注册到 capabilities/tools/，通过现有 CapabilityRegistry 自动发现。
