① 三层记忆总览

三层记忆各有作用域和生命周期。 已实现 WorkingMemory / EpisodicMemory 已有； 本期新增 SemanticMemory 是核心建设目标； 延期 Sub-Agent 独立记忆后续按需扩展。

设计原则：不引入新基础设施依赖。个人规模（单 Owner，数百到数千条记忆）SQLite FTS5 全文检索毫秒级响应，向量 DB 属过度设计。

② 数据模型

新增两张表：memories（语义记忆主表）和 memory_keys（entity_key 动态注册表）。

Upsert 冲突策略：有 entity_key → 按 key 查找已有记录 → 新值 confidence > 旧值 × 0.8 则覆盖；explicit 永远覆盖 inferred。无 entity_key（NULL）→ 直接 INSERT，不做去重。

③ entity_key 混合策略

混合方案：预置 key 走结构化 Upsert（保证去重），超出范围的走自由文本 Insert（保留完整信息）。

自扩展机制：ConsolidationWorker 每次运行都从 memory_keys 查完整列表传给 LLM，LLM 提出的新 key 写回后，下次运行自动包含——key 库随使用不断丰富，不需要人工维护代码。

④ 写入流水线

两条路径：路径 A 用户主动触发（精准高置信）； 路径 B Session 沉淀（后台异步，含 memory_keys 动态扩展）。

不做每 turn 提取：噪音大且额外消耗 LLM API。只在 session 自然结束后整体提取一次，性价比更高。

⑤ 读取流水线

被动注入（每次 turn 自动）覆盖高频常识；主动查询（Agent 按需）覆盖低频深查。

注入位置：与 _session_todos_section() 相同模式，在 _stream_inner() 开始时动态拼接，每次 turn 刷新。当 memory_enabled=False 时此方法直接返回空字符串。

⑥ 与 Agent Loop 集成

最小侵入原则：4 个 hook，不改 AgentLoop 核心逻辑。Sub-Agent 记忆延期，本期 hook 只对 Sebastian 主管家生效。

sequenceDiagram participant U as 用户 participant BA as BaseAgent participant SM as SemanticMemory participant AL as AgentLoop participant EB as EventBus participant CW as ConsolidationWorker U->>BA: run_streaming(message, session_id) note over BA: Hook 1 — 被动注入 BA->>SM: top_memories(goal, limit=8) SM-->>BA: memories[] BA->>BA: effective_prompt = system + memory_section + todos BA->>AL: loop.stream(effective_prompt, messages) loop 多轮工具调用 AL-->>BA: ToolCallReady alt memory_save note over BA: Hook 2 — 即时写入 BA->>SM: upsert(source=explicit) else memory_search note over BA: Hook 3 — 主动查询 BA->>SM: search(query) SM-->>BA: results[] end end BA->>BA: TurnDone 持久化对话 note over EB,CW: Hook 4 — 异步沉淀 EB-->>CW: SESSION_IDLE or SESSION_STALLED CW->>SM: get_all_keys() CW->>CW: LLM 提取事实 CW->>SM: upsert_keys(new_keys) CW->>SM: upsert_memories(facts)
Hook文件改动说明
1 被动注入 core/base_agent.py _stream_inner() 加 _memory_section()，仿 todos 模式
2 即时写入 capabilities/tools/memory_save.py 新增 @tool，LOW 权限，直接调 SemanticMemory.upsert()
3 主动查询 capabilities/tools/memory_search.py 新增 @tool，FTS5 查询返回 ToolResult
4 Session 沉淀 memory/consolidation.py ConsolidationWorker 订阅 SESSION_IDLE / STALLED 事件
