① 系统层次总览

记忆系统通过 BaseAgent（基础 Agent）接入运行时：_memory_section() 负责每轮自动注入， memory_save / memory_search 提供显式工具入口。读取与写入走独立管线； Session Consolidation（会话沉淀）由 SESSION_COMPLETED 异步触发，产出的候选记忆必须回流统一写入流水线，不能直接修改存储。

BaseAgent 是运行时接入点：自动注入 hook 与显式 memory 工具都从这里进入
读取与写入管线分离；读取只 fetch，写入统一 Resolve / Persist / Decision Log
Session Consolidation 是异步侧车，只输出候选 artifacts / actions 并回流写入管线
Cross-Session、Maintenance、Pending Candidates 属于后续独立 spec

② 写入流水线（Write Pipeline）完整视图

四类来源（显式写入、对话推断、行为观察、沉淀派生）统一走同一条流水线。 LLM（大语言模型）只负责语义提炼（Semantic Extraction），最终 ADD / SUPERSEDE / MERGE / EXPIRE / DISCARD 决策由 Resolve（冲突解析）层统一输出。

③ 读取路径（Read Path）与四条检索通道（Retrieval Lane）

④ 后台沉淀（Background Consolidation）触发链

后台沉淀不是 BaseAgent 内部 hook（钩子），由 SESSION_COMPLETED（会话完成）事件驱动。 Consolidator（沉淀器）产出的新 Artifact（记忆产物）必须再次进入统一写入流水线，不能绕过 Resolve（冲突解析）直接落库。 ⚠️ Cross-Session Consolidation 和独立 Maintenance Worker 已推迟。

⑤ MemoryArtifact（记忆产物）生命周期状态机

记忆不是 KV（键值）配置项。事实更新默认保留历史轨迹（旧记录 status → superseded（已取代））， 让系统既能查当前真值（Current Truth），也能回溯历史变更轨迹。 ⚠️ Summary Replacement（摘要替换）策略尚未 spec 化；当前 summary 为 append-only。
